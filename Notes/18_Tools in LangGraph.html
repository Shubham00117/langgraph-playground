Topic 127. Core Concepts
Concept
Tool: Python function + @tool. LLM uses docstrings to determine when and how to call them.
Tool Calling: The LLM identifies the intent to use a tool and outputs its name + structured arguments instead of raw text (execution happens later in ToolNode).
ToolNode: The execution layer that runs the requested Python functions and returns results as ToolMessage objects, feeding them back into the graph state.
tools_condition: The decision logic that inspects the LLM's response; if tool calls are present, it loops to ToolNode, otherwise it terminates the workflow.

Topic 128. Graph Architecture
Design
Entry: Workflow begins at __start__ and moves immediately to the chat_node.
Decision Edge: The tools_condition acts as a router after the LLM responds.
Branching: If no tool is needed, it goes to __end__. If a tool is requested, it routes to the tools node.
The Loop: After execution, the tools node cycles back to chat_node to provide the LLM with the tool output.

Topic 130. Bind Tools & Wire the Graph
Code
bind tools to llm 
# Collect all tools into a list
tools = [get_stock_price, search_tool, calculator]

# Inject tool schemas into LLM context
llm_with_tools = llm.bind_tools(tools)

# State definition
class ChatState(TypedDict):
    messages: Annotated[list[BaseMessage],
                          add_messages]
bind_tools() injects schemas into the LLM, enabling tool selection.
graph wiring 
# Node: LLM (may answer or call tool)
def chat_node(state: ChatState):
    response = llm_with_tools.invoke(state['messages'])
    return {"messages": [response]}

# Node: executes tool calls
tool_node = ToolNode(tools)

# Build graph
graph = StateGraph(ChatState)
graph.add_node("chat_node", chat_node)
graph.add_node("tools",     tool_node)

graph.add_edge(START, "chat_node")
graph.add_conditional_edges("chat_node", tools_condition)
graph.add_edge("tools", "chat_node")  # loop back

chatbot = graph.compile()

Topic 131. Streamlit Frontend ‚Äî Streaming with Tool Status
Frontend
1
Use stream_mode="messages" for token-by-token updates.
2
Distinguish between **AIMessage** (text) and **ToolMessage** (status).
3
Use st.status() for tool progress and st.write_stream() for text.
streamlit_frontend_tool.py 
with st.chat_message("assistant"):
    status_holder = {"box": None}

    def ai_only_stream():
        for chunk, _ in chatbot.stream(
            {"messages": [HumanMessage(content=user_input)]},
            config=CONFIG,
            stream_mode="messages"
        ):
            # ToolMessage ‚Üí show status box
            if isinstance(chunk, ToolMessage):
                name = getattr(chunk, "name", "tool")
                if status_holder["box"] is None:
                    status_holder["box"] = st.status(
                        f"Using `{name}` ...", expanded=True)
                else:
                    status_holder["box"].update(
                        label=f"Using `{name}` ...",
                        state="running")

            # AIMessage ‚Üí stream text to user
            if isinstance(chunk, AIMessage):
                yield chunk.content

    ai_msg = st.write_stream(ai_only_stream())
