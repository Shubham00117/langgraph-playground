Reducers

Reducers define how node updates merge with state. Each key can have a custom strategy: replace, add, or merge.
Three Reducer Strategies
Replace (default)
Overwrites old values. Used for scalars like scores and strings.
Add / Append
Appends updates to a list via Annotated[list, add]. For accumulation across rounds.
Merge / Custom
Custom functions to combine data (e.g., dict merging or taking max/min).
  • â†’Conflict Resolution: Resolves overlapping writes from parallel nodes safely.
  • â†’History Retention: Ensures feedback is accumulated across multiple evaluation rounds.

LangGraph Execution Model

Execution follows a Pregel-style message-passing model, proceeding in discrete super-steps (rounds) until work is complete.
1
Compilation
Registers nodes/edges and validates the graph structure before execution.
2
Injection
User values populate the initial state snapshot.
3
Activation
Entry nodes receive initial messages and become active.
4
Super-Steps
All active nodes run in parallel, process state, and return updates (messages).
5
Propagation
Updates flow via edges. Nodes receiving messages activate for the next round; others remain idle.
6
Halting
Stops when no nodes are active and no messages are in transit.

Key insight: Parallelism is native. Multiple active nodes in a round run simultaneously by defaultâ€”no special "parallel code" required.
