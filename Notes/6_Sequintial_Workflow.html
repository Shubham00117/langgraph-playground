33 Core Concepts

What is LangGraph?

LangGraph is a library for building stateful, graph-based workflows with LLMs. Each workflow is a directed graph where nodes perform computation and edges define execution order.

StateGraph Nodes (functions) Edges (flow) State (shared data)
34 Graph Anatomy

How a workflow is structured

START → node_1 → node_2 → END
  • ①Define a State class (TypedDict) — holds all data shared between nodes
  • ②Write node functions — take state as input, update it, return state
  • ③Create a StateGraph, add nodes & edges
  • ④Compile the graph → workflow
  • ⑤Invoke with initial state → returns final state
35 Example 1 — BMI Calculator

Pure Python workflow (no LLM)

START → calculate_bmi → label_bmi → END 
# State
class BMIState(TypedDict):
    weight_kg: float
    height_m:  float
    bmi:       float
    category:  str

# Node 1 – compute BMI
def calculate_bmi(state: BMIState) -> BMIState:
    bmi = state['weight_kg'] / (state['height_m']**2)
    state['bmi'] = round(bmi, 2)
    return state

# Node 2 – assign label
def label_bmi(state: BMIState) -> BMIState:
    bmi = state['bmi']
    if bmi < 18.5:   state['category'] = "Underweight"
    elif bmi < 25:  state['category'] = "Normal"
    elif bmi < 30:  state['category'] = "Overweight"
    else:           state['category'] = "Obese"
    return state

# Build & run
graph = StateGraph(BMIState)
graph.add_node('calculate_bmi', calculate_bmi)
graph.add_node('label_bmi', label_bmi)
graph.add_edge(START, 'calculate_bmi')
graph.add_edge('calculate_bmi', 'label_bmi')
graph.add_edge('label_bmi', END)
workflow = graph.compile()

result = workflow.invoke({'weight_kg': 80, 'height_m': 1.73})
# → {'weight_kg':80, 'height_m':1.73, 'bmi':26.73, 'category':'Overweight'}
36 Example 2 — LLM Q&A Workflow

Single-node LLM workflow

START → llm_qa → END 
class LLMState(TypedDict):
    question: str
    answer:   str

model = ChatOpenAI()

def llm_qa(state: LLMState) -> LLMState:
    question = state['question']
    prompt   = f'Answer the following question {question}'
    answer   = model.invoke(prompt).content
    state['answer'] = answer
    return state

# Result for "How far is moon from the earth?"
# → "The average distance … is about 384,400 km (238,855 miles)."
37 Example 3 — Prompt Chaining

Multi-step LLM pipeline — Blog Generator

Each LLM call feeds its output into the next, building up the state progressively.

START → create_outline → create_blog → END 
class BlogState(TypedDict):
    title:   str
    outline: str
    content: str

def create_outline(state: BlogState) -> BlogState:
    prompt = f'Generate a detailed outline for a blog on - {state["title"]}'
    state['outline'] = model.invoke(prompt).content
    return state

def create_blog(state: BlogState) -> BlogState:
    prompt = f'Write a detailed blog on {state["title"]} using the outline\n{state["outline"]}'
    state['content'] = model.invoke(prompt).content
    return state

# invoke({'title': 'Rise of AI in India'})
# → state contains title + outline + full blog content
38 Key Takeaways

LangGraph vs LangChain

LangGraph builds on LangChain but uses an explicit graph model — better for complex, stateful, multi-step agents. LangChain = sequential chains. LangGraph = graph of nodes.

State is Central

The TypedDict state is the single source of truth. Every node reads from it and writes back to it. Final workflow.invoke() returns the fully-populated state.

graph = StateGraph(State) → add_node → add_edge → compile() → invoke(initial_state)
