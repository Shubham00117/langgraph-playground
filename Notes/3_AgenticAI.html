Definition

Agentic AI is a type of AI that can take up a task or goal from a user and then work toward completing it on its own, with minimal human guidance. It plans, takes action, adapts to changes, and seeks help only when necessary.

GEN AI

Responds to one prompt at a time. Waits for your next instruction after every response.

‚ö° Reactive

AGENTIC AI

Takes one goal and autonomously drives itself to completion across many steps.

STEP 03

STEP 04

8 applications ‚Üí parses resumes ‚Üí 2 strong, 3 partial, 3 weak ‚Üí asks to schedule top 2.

STEP 05

Scheduling

Checks calendar ‚Üí Friday free ‚Üí drafts & sends 45-min interview invite email to candidates.

STEP 06

Interviewing

Sends reminder (2 interviews today) + emails interviewer a list of role-specific questions.

STEP 07

Offer Letter

HR picks 1 candidate ‚Üí agent drafts offer letter ‚Üí HR approves ‚Üí sends and tracks acceptance.

STEP 08

Onboarding

Candidate accepts ‚Üí welcome email sent, IT access submitted, laptop provisioned, intro meetings scheduled.

Autonomy

The AI system's ability to make decisions and take actions on its own to achieve a given goal, without needing step-by-step human instructions. It is proactive, not just reactive.

AUTONOMY OPERATES IN THREE FACETS

Execution ‚Äî carries out tasks independently
Decision Making ‚Äî chooses next action from options
Tool Usage ‚Äî selects and calls the right tools

AUTONOMY CAN BE CONTROLLED

Permission ScopeLimit what actions the agent can do alone. (Can screen, but needs approval to reject anyone.)
HITL (Human-in-the-Loop)Insert checkpoints requiring human sign-off. ("Can I post this JD?")
Override ControlsUser can stop, pause, or change agent behaviour at any time.
Guardrails / PoliciesHard rules the agent must always follow. (Never schedule interviews on weekends.)
‚ö† Autonomy can be dangerous ‚Äî real risks Sends offers with incorrect salaries ¬∑ Shortlists by age or nationality (discrimination) ¬∑ Overspends on LinkedIn ads

Goal Oriented

The AI operates with a persistent objective in mind and continuously directs all its actions to achieve that objective, rather than just responding to isolated prompts.

Goals act as a compass for Autonomy ‚Äî every decision is measured against the goal
Goals can come with constraints ‚Äî remote-only, budget limit, specific tech stack
Goals are stored in core memory ‚Äî as a JSON object (conceptual representation)
Goals can be altered ‚Äî user can pivot mid-task (e.g. hire freelancer instead)

Planning

The agent's ability to break down a high-level goal into a structured sequence of actions or subgoals and decide the best path to achieve the desired outcome.

S1
Generate Multiple Candidate Plans Plan A: Post JD on LinkedIn, GitHub Jobs, AngelList ¬∑ Plan B: Internal referrals and hiring agencies
S2
Evaluate Each Plan Criteria ‚Äî Efficiency ¬∑ Tool Availability ¬∑ Cost ¬∑ Risk ¬∑ Alignment with constraints (remote-only? budget?)
S3
Select the Best Plan Via HITL input ("Which option do you prefer?") or pre-programmed policy ("Favor low-cost channels first")

üìå Planning is iterative ‚Äî the agent re-plans during execution if a tool fails or results change.

Reasoning

Reasoning is the cognitive process through which the agent interprets information, draws conclusions, and makes decisions ‚Äî both while planning ahead and while executing actions in real time.

DURING PLANNING
Goal DecompositionBreak abstract goals into concrete steps

Adaptability

The agent's ability to modify its plans, strategies, or actions in response to unexpected conditions ‚Äî all while staying aligned with the goal.

Failures ‚Äî Calendar API down ‚Üí switch to email-based scheduling
External Feedback ‚Äî Only 2 applicants ‚Üí broaden JD scope to full-stack
Changing Goals ‚Äî Hiring a full-time engineer ‚Üí switching to a freelancer

Context Awareness

The agent's ability to understand, retain, and utilize relevant information from the ongoing task, past interactions, user preferences, and environmental cues to make better decisions throughout a multi-step process.

TYPES OF CONTEXT

Original GoalThe main objective driving all actions
Interaction HistoryJD finalized and posted to LinkedIn & GitHub Jobs
Environment StateApplicants so far = 8, LinkedIn promo ends in 2 days
Tool ResponsesResume parser ‚Üí "Candidate B has 3y Django + AWS experience"
User PreferencesPrefers remote-first, likes questions in a Google Doc
GuardrailsDo not send offer without explicit user approval
Implemented through Memory ‚Äî the agent stores and retrieves context
Short-term memory ‚Üí current session context (active conversation)
Long-term memory ‚Üí persists across sessions
