Topic 123. What is a Thread?

Concept
A Thread is a collection of individual traces linked together to represent a single, continuous conversation.
Allows monitoring the entire multi-turn chat history logically within LangSmith instead of viewing isolated runs.

Topic 124. Linking Traces & The CONFIG Object

Implementation
A unique identifier (e.g., UUID) must be passed through a metadata key (thread_id, session_id, or conversation_id).
On every turn, pass a CONFIG dictionary to link the trace into the correct thread:
# Python ‚Äî Streamlit Example
CONFIG = {
    "configurable": {"thread_id": st.session_state["thread_id"]}, # Used by LangGraph for state/memory
    "metadata": {"thread_id": st.session_state["thread_id"]},     # Used by LangSmith to group traces
    "run_name": "chat_turn",                                      # Labels run for easy filtering
}

Topic 125. End-to-End Flow

1. Generate & store UUID ‚Üí 2. User sends message ‚Üí 3. Build CONFIG ‚Üí 4. chatbot.stream(..., config=CONFIG)
LangGraph updates conversation memory, and LangSmith groups the trace under the thread.
Switching conversations simply involves loading a different thread_id.

Topic 126. Observability UI in LangSmith

Interface
The Threads tab gives a clean, high-level conversation view.
The traditional Runs tab shows low-level, individual trace details.
Threads display turns side-by-side, track token counts, latency, cost, and the entire call hierarchy.
