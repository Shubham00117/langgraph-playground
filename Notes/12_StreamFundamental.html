Topic 75
Execution Model
üîÅ Script reruns top-to-bottom on every interaction

Every time a user interacts (clicks, types, selects), Streamlit reruns your entire script from top to bottom. This is the most critical concept to internalize.

‚ö†Ô∏è Expensive operations (DB calls, model loads) placed at the top level will re-execute on every click ‚Äî always wrap them in @st.cache_data or @st.cache_resource.

Topic 79
Session State
Persist data across reruns ‚Äî per user, per session

st.session_state is a dict that survives reruns within the same browser session. Different users each get their own isolated state.

# Initialize (always guard with 'not in')
if "count" not in st.session_state:
    st.session_state["count"] = 0

# Read ‚Äî two equivalent syntaxes
x = st.session_state["count"]     # dict style
x = st.session_state.count          # attribute style

# Write
st.session_state["count"] += 1

# Delete
del st.session_state["count"]

# Widget key ‚Üí auto-syncs to session_state
st.text_input("Name", key="username")
st.write(st.session_state.username)  # always in sync
Callbacks ‚Äî run before the script reruns

Attach on_click / on_change to widgets for event-driven logic.

def increment():
    st.session_state.count += 1

st.button("‚ûï Add", on_click=increment)
st.write("Count:", st.session_state.count)
üí° session_state vs cache: session_state is per-user, per-session. Cached values are shared across all users.

Topic 80
Caching ‚Äî Critical for Performance
Two decorators ‚Äî know when to use each
@st.cache_data

Returns data copies. Use for DB queries, API calls, dataframe transforms, ML inference results.

@st.cache_data
def load_data(url):
    return pd.read_csv(url)
@st.cache_resource

Returns shared object. Use for ML models, DB connections, LangGraph workflows.

@st.cache_resource
def load_model():
    return build_chatbot()
Cache parameters
# TTL ‚Äî auto-expire after N seconds (e.g. live data)
@st.cache_data(ttl=300)          # expire every 5 min
def get_live_prices(): ...

# Max entries ‚Äî limit memory
@st.cache_data(max_entries=50)
def query_db(user_id): ...

# Persist to disk across restarts
@st.cache_data(persist="disk")
def expensive_computation(): ...

# Manual cache clear
if st.button("Clear cache"):
    st.cache_data.clear()
‚ö†Ô∏è @st.cache_resource returns the actual shared object ‚Äî if you mutate it (e.g., append to a list), all users see that change. Use @st.cache_data when you don't want mutations shared.

Topic 81
Chat UI (LLM / Chatbot Pattern)
Complete production chatbot pattern
import streamlit as st
from langchain_core.messages import HumanMessage

# 1. Load model once ‚Äî cached globally
@st.cache_resource
def get_chatbot():
    return build_langgraph_chatbot()

chatbot = get_chatbot()

# 2. Init chat history once per session
if "messages" not in st.session_state:
    st.session_state.messages = []

# 3. Render all past messages
for msg in st.session_state.messages:
    with st.chat_message(msg["role"]):
        st.markdown(msg["content"])

# 4. Get new user input
if prompt := st.chat_input("Ask me anything..."):

    # 5. Show + save user message
    st.session_state.messages.append(
        {"role": "user", "content": prompt}
    )
    with st.chat_message("user"):
        st.markdown(prompt)

    # 6. Call LLM + stream response
    with st.chat_message("assistant"):
        placeholder = st.empty()
        full = ""
        for chunk in chatbot.stream({"messages": [HumanMessage(prompt)]}):
            full += chunk
            placeholder.markdown(full + "‚ñå")
        placeholder.markdown(full)

    # 7. Save AI response
    st.session_state.messages.append(
        {"role": "assistant", "content": full}
    )
‚úÖ Use st.markdown() inside chat_message instead of st.text() ‚Äî it renders bold, code blocks, lists from LLM output properly.

Forms ‚Äî Batch Widget Submissions
Group widgets ‚Äî script only reruns on submit

Without forms, every keypress/slider move triggers a full rerun. Forms batch all changes and rerun only when submitted ‚Äî essential for search or settings panels.

with st.form("search_form"):
    query  = st.text_input("Search query")
    top_k  = st.slider("Results", 1, 20, 5)
    source = st.selectbox("Source", ["Web", "DB"])
    submitted = st.form_submit_button("üîç Search")

if submitted:
    results = search(query, top_k, source)
    st.dataframe(results)

Topic 84
Secrets Management
Never hardcode API keys ‚Äî use secrets.toml

Streamlit reads .streamlit/secrets.toml locally and from the Streamlit Cloud dashboard in production.

# .streamlit/secrets.toml  (never commit to git!)
OPENAI_API_KEY = "sk-..."
GROQ_API_KEY   = "gsk-..."

[database]
host     = "db.example.com"
password = "secret123"
# Access in your Python code
api_key  = st.secrets["OPENAI_API_KEY"]
db_pass  = st.secrets["database"]["password"]

# Or as attributes
api_key  = st.secrets.OPENAI_API_KEY
‚ö†Ô∏è Add .streamlit/secrets.toml to .gitignore. In production, set secrets via Streamlit Cloud's UI ‚Äî they're injected automatically.

Topic 85
Multipage Apps
Automatic page routing via directory structure

Files inside a pages/ folder become separate pages automatically ‚Äî no routing code needed.

my_app/
‚îú‚îÄ‚îÄ app.py               # Home page
‚îî‚îÄ‚îÄ pages/
    ‚îú‚îÄ‚îÄ 1_üìä_Dashboard.py
    ‚îú‚îÄ‚îÄ 2_üí¨_Chat.py
    ‚îî‚îÄ‚îÄ 3_‚öôÔ∏è_Settings.py
‚úÖ Prefix filenames with numbers to control sort order. Emojis in names appear in the sidebar navigation automatically.

Share data between pages with st.session_state ‚Äî it persists across page navigation within the same session.

# In app.py (Home)
st.session_state["user_id"] = "abc123"

# In pages/2_Chat.py ‚Äî same session, same state
user = st.session_state.get("user_id", "guest")
st.write(f"Chatting as: {user}")
